In summary, we are developing a system to help companies comply with inspections, not necessarily DOT inspections. The system has a device and this web app. Operators use the device to perform inspections and submit records.

So finally the device is ready.

today we will make the endpoints to configure and upload inspections.

Lets start with the endpoit to upload inspections.

The device is not Android or anything fancy; the competitive advantage is that it is super cheap and maintenance-free. It runs in an STM32 chip, an ARM m7 core, so it is very simple.

So specs:

We need a REST endpoint where the device can simply open a socket and make a post. No SSL, no security for now, the device will use a server address and make a POST like this:

explanation:

So the data is inspired in EDI, but of course it is not edi, its my format and we can tune it if we need to.

Basically, it is a collection of rows. Each row has a prefix indicating the data type, and fields are separated by "*".
Fields will always come in the same order or it is an error. As mentioned, we might need to add fields to the db, and if there are heavy discrepancies, let me know before making complex fixes.
